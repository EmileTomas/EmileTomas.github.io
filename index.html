语义分析(chapter 5)

实验目的

通过符号表的构建,对Parsing 输出的抽象语法树进行类型检查.

本实验虎书并未提供现成框架,整个Semant package需要自己建立,当然虎书在pdf中已经提供了大部分的思路和实例代码了.主要工作便是将示例代码推广以及解决其中遇见的各种特殊情况.相比与词法分析与语法分析需要对课本有更深刻的掌握.

实验说明

实现说明

整体思路:

将语法分析输出的抽象语法输入至语义分析,在语义分析的过程中,构建符号表,一边进行符号表的增添,一边进行类型检查,当发现类型不匹配,便输出错误.

  1. 符号表采用Imperative Symbol Table ,通过建立一个undo栈实现不同环境的保存.

  2. 整个分析过程有两个环境(符号表实例),分别是Tenv和Venv.

    • Tenv: Type Environment, 用于记录Type项的相关信息
    • Venv:Value Environment, 用于记录Function以及Variables

  3. 类型检查有三个部分,分别为:

    • Type-check expression
    • Type-check variable subscript, and fields
    • Type-check Declaration
  4. 修改Main.java进行语义分析

Guide to GPU Structure

If you once used GPU to do some generally purpose programming, you must be astonished by the fast speed of GPU. Generally speaking, GPU can provide a speed boost about 200 to 300 times over a CPU-bound job. So I will tell you how this happen.

GPU is much faster than CPU

At the beginning, GPU was introduced to speed up the graphics processing with fixed function which provide a powerful floating point calculation ability. So people wandering if it’s possible to make use of GPU to do some calculation-heavy task. In the early time, people can only use assemble language to do programming on GPU. Wherever there is a need, there will be a supply. GPU manufacturers also tried to make their GPU more useful. For example, NVIDIA provide a C-based language named CUDA which is widely used in the AI, Machine Learning area.

Why GPU is so fast

To understand that, we need know the architecture of GPU at first. Figure below shows the Architecture of G200 (2009)

  • GTX 200 contains 10 TPC(Texture Processing Cluster)
  • Each TPC consists of 3 SM (Streaming Multiprocessors)
  • Each SM consists of 8 SP (Stream Processors, also called shader processor or CUDA core )
  • Each SM supports up to 1024 threads

So there are 8*3*10=240 stream processors! That’s the reason why GPU is so fast. But how could we put so many core into such a limited area (the chip size of CPU and GPU is almost same)?

Comparing the structure of CPU and GPU you can find the most area of the CPU is used for Cache.

How to programming with GPU

The structure of GPU brings powerful parallel performance to GPU. And there are two choice to programming on GPU, using CUDA and OpenCL.

CUDA (compute Unified Device Architecture)

CUDA was firstly introduced to Geforce 8800 by NVIDIA in 2006. It’s also the name of the corresponding programming language which is based on C.

One Recent GPU – GTX 850M

  • GTX 850M contains 5 SM
  • Each SM consists of 128 CUDA cores(SP)
